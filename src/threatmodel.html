Threat Model

Our threat model shows the architecture of the system. Based on this, threats to the system or the user are identified and countermeasures are defined.

Description

To be able to use the application, the user must initially have a browser and an Internet connection. With the connection to the web server, the user crosses the first trust boundary, which is the Internet provider. The user then connects to the web server via https. The trust boundary of the web server has to be crossed for this purpose. The web server provides the stored HTML, CSS and Javascript files and delivers them to the user. The user's browser loads them into memory and executes them locally.

Threats

Denial of Service

The application could become the target of a DDOS attack. As a result, accessibility is not guaranteed. Depends on the availability of the Internet on the side of the user and the server. Availability varies depending on the host system.

Tampering

Manipulation of the web server by an attacker, so that incorrect files are provided or changed. Can lead to information disclosure.

Mitigations

  • the web server should be protected against attacks as best as possible.
  • ensure availability through DDOS protection and backup line.
  • offline functionality of the web application, thus not dependent on the internet connection.
  • ensure integrity of the application by hashing, so that the user can verify integrity himself.
